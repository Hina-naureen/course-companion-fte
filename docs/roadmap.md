# Development Roadmap – Course Companion FTE

## Timeline: Hackathon Sprint (3 Phases)

---

## Phase 1 – Zero-Backend-LLM Core
**Goal:** A working ChatGPT Custom GPT backed by a real FastAPI + PostgreSQL API.

### Sprint 1: Foundation (Day 1)
- [ ] PostgreSQL setup with Alembic migrations
- [ ] User auth: register, login, JWT issue/verify
- [ ] Seed 3 free chapters + chapter 4 (pro-locked) to DB
- [ ] `GET /chapters` list endpoint with tier-aware locking
- [ ] `GET /chapters/{id}` with 403 on tier mismatch

### Sprint 2: Quiz & Progress (Day 1–2)
- [ ] Seed MCQ quizzes for chapters 1–3
- [ ] `GET /chapters/{id}/quiz` — questions without answers
- [ ] `POST /quizzes/{id}/attempt` — score + explanations
- [ ] `GET /progress` + `PUT /progress/chapters/{id}`

### Sprint 3: Search + Rate Limiting (Day 2)
- [ ] `GET /search` with pg_tsvector
- [ ] Search rate limit: 10/day for free tier (search_log table)
- [ ] Quiz attempt rate limit: 3/day for free tier

### Sprint 4: ChatGPT Integration (Day 2–3)
- [ ] Write OpenAPI 3.1 spec for ChatGPT Actions (auto-generated by FastAPI)
- [ ] Configure Custom GPT with system prompt
- [ ] System prompt: conversational course guide, quiz proctor, freemium upsell
- [ ] End-to-end test: enroll → read chapter → take quiz → check progress

### Phase 1 Deliverable
- Working ChatGPT GPT that teaches GenAI fundamentals
- Demo: user registers, reads chapters, takes quiz, sees progress

---

## Phase 2 – Hybrid Intelligence
**Goal:** Add Claude-powered skills that enhance the learning experience.

### Sprint 5: Skills Layer (Day 3–4)
- [ ] `POST /skills/explain` — multi-depth concept explainer
- [ ] `POST /skills/hint` — adaptive quiz hints (3 levels)
- [ ] `POST /skills/quiz/generate` — novel question generation (Pro)
- [ ] `POST /skills/search/semantic` — rerank search results
- [ ] Per-user daily limits for free tier (5 explains/day)

### Sprint 6: Streaming & UX (Day 4)
- [ ] Server-sent events (SSE) for streaming Claude responses
- [ ] Graceful fallback when Claude API unavailable (serve static explanation)
- [ ] Skills endpoints integrated into ChatGPT GPT actions

### Phase 2 Deliverable
- GPT can now explain any concept at user's level
- GPT provides hints without spoiling quiz answers
- Pro users get AI-generated practice questions

---

## Phase 3 – Full Web App
**Goal:** Replace ChatGPT frontend with a custom Next.js app.

### Sprint 7: Next.js Scaffold (Day 4–5)
- [ ] Next.js 15 with App Router + TypeScript
- [ ] Tailwind CSS + shadcn/ui component library
- [ ] Auth pages: login, register, JWT in httpOnly cookie
- [ ] Protected route middleware

### Sprint 8: Core Pages (Day 5–6)
- [ ] `/dashboard` — progress ring, streaks, next chapter CTA
- [ ] `/chapters` — chapter library with lock icons
- [ ] `/chapters/[slug]` — Markdown reader with estimated time
- [ ] `/quizzes/[id]` — Interactive MCQ + open-ended quiz engine

### Sprint 9: AI-Enhanced Pages (Day 6)
- [ ] "Explain this" button → streaming Claude explanation sidebar
- [ ] Adaptive hint system in quiz UI
- [ ] Semantic search bar with reranked results

### Sprint 10: Pro Features (Day 6–7)
- [ ] `/progress/analytics` — charts, streaks, weakness heatmap (Pro)
- [ ] `/upgrade` — pricing page with Stripe checkout
- [ ] AI quiz generation UI for Pro users

### Phase 3 Deliverable
- Full web app replacing ChatGPT frontend
- Complete freemium flow with Stripe upgrade path

## Architecture Decisions Log

| Decision | Choice | Rationale |
|---|---|---|
| DB | PostgreSQL | JSONB + pg_tsvector covers all Phase 1 needs |
| Auth | JWT HS256 | Stateless, works cleanly with ChatGPT Actions |
| LLM (Phase 2) | claude-sonnet-4-6 | Best intelligence/latency for edu tasks |
| Rate limit store | DB (Phase 1) → Redis (later) | DB sufficient for hackathon scale |
| Search | pg_tsvector → pgvector (later) | Keyword search first, semantic later |
| Frontend | ChatGPT GPT → Next.js | Ship fastest UX first, own it in Phase 3 |
